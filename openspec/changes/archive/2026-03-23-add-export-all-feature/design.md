## Context

系统已具备 Excel 导入功能（`GET /api/rule-config/template` 和 `POST /api/rule-config/import`），使用 `xlsx` 库处理 Excel 文件。本次需要新增导出功能，与导入功能形成闭环。

## Goals / Non-Goals

**Goals:**
- 提供一键导出所有规则配置的功能
- 导出格式与导入模板完全兼容
- 支持大数据量导出（万级数据）
- 文件名包含时间戳，便于区分

**Non-Goals:**
- 不支持按条件筛选后导出（导出全部）
- 不支持其他格式（CSV、JSON 等）
- 不支持定时自动导出

## Decisions

### 1. 数据查询方式：流式查询 vs 一次性查询

**决策**: 使用一次性查询（`SELECT * FROM rule_config`）

**理由**:
- 规则配置数据量通常在万级以内，一次性查询内存可承受
- 实现简单，与现有 `getList` 逻辑保持一致
- 如需优化，后续可添加分页查询再合并

**替代方案**: 流式查询适合百万级数据，但实现复杂

### 2. Excel 生成方式：服务端生成 vs 前端生成

**决策**: 服务端生成 Excel 文件

**理由**:
- 与现有导入模板生成逻辑一致（`template` 接口）
- 前端无需额外依赖
- 便于后续添加权限控制

**替代方案**: 前端生成可减少服务端压力，但需要引入 xlsx 库

### 3. 文件命名格式

**决策**: `rule_config_export_YYYYMMDD_HHMMSS.xlsx`

**理由**:
- 包含日期时间，避免文件覆盖
- 语义清晰，易于识别
- 使用本地时间（+08:00）

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 大数据量导出内存溢出 | 当前数据量可控，如有需要后续改为流式生成 |
| 导出时数据被修改 | 当前为单用户系统，风险低；后续可添加导出锁定 |

## Migration Plan

无需迁移，纯功能新增。
